## 搬瓦工 KVM 一键加速是什么意思

先说清楚概念。

**搬瓦工 KVM 一键加速**，指的是在 KVM 架构的 VPS 上，通过更换内核或安装加速模块，把默认的 TCP 拥塞控制算法换成对高延迟、高丢包环境更友好的方案。常见的有：

- **BBR**（Google 出品）：主流选择，现代 Linux 内核基本原生支持，装了就能用
- **BBRplus**（BBR 魔改版）：在 BBR 基础上做了调整，高丢包环境表现更激进一些
- **BBR2**：Google 内部迭代版本，整体上比原版 BBR 更稳
- **Lotserver（锐速）**：Appex 商业加速方案，效果好但需要换内核，操作稍复杂

搬瓦工全站用 KVM 架构，对内核替换没有限制。这是 OpenVZ 做不到的事。

## 路径一：KiwiVM 后台直接装带 BBR 的系统（最省事）

不想折腾命令行，就用这个方法。

搬瓦工自带的 KiwiVM 控制面板支持一键重装系统，其中有带 `bbr` 后缀的镜像选项，装完就自带 BBR 加速，连配置都不用动。

**操作步骤如下：**

1. 登录搬瓦工账户，进入 Services → My Services，点击对应 VPS 的 KiwiVM Control Panel
2. 左侧菜单找到 **Install new OS**
3. 在系统列表里找带 `bbr` 后缀的版本，比如 `debian-12-x86_64-bbr` 或 `ubuntu-22.04-x86_64-bbr`
4. 注意：`_64` 结尾是 64 位系统，优先选这个
5. 点击 **Reload**，等待重装完成（通常几分钟内）
6. 重装完成后记录新的 SSH 端口和 root 密码

重装前记得备份数据。

用 SSH 登录之后，验证 BBR 是否生效：

bash
lsmod | grep bbr


看到 `tcp_bbr` 就说明装好了。

这条路的优点是零命令行操作，适合刚买完机器的新手直接用。缺点是换完系统一切从零开始，原有的服务需要重新配置。

## 路径二：一键脚本安装 BBRplus / 锐速 / BBR2（效果更强）

已经有在跑的服务，不想重装系统，就走这条路。

这个方法需要用 SSH 登录到 VPS，然后跑一个多合一加速脚本，里面包含 BBR、BBRplus、锐速（Lotserver）、BBR2 等方案，选哪个装哪个。

**前置准备：**

CentOS 系统先跑这行：

bash
yum install ca-certificates wget -y && update-ca-trust force-enable


Debian / Ubuntu 系统跑这行：

bash
apt-get install ca-certificates wget -y && update-ca-certificates


**安装加速脚本（推荐不卸载内核版本）：**

bash
wget -O tcpx.sh "https://git.io/JYxKU" && chmod +x tcpx.sh && ./tcpx.sh


回车之后会出现选择菜单，通常是这些选项：

- 1：BBR
- 2：BBR2
- 3：BBR魔改版
- 4：BBRplus
- 5：BBRplus 新版内核
- 6：Lotserver 锐速
- 7：ENET BBR 魔改版

**怎么选？**

普通用户直接选 BBRplus（4 或 5）。效果比原版 BBR 好，操作也不复杂。

追求极致的可以试试 Lotserver 锐速，在丢包严重的线路上效果明显，但需要换内核，操作有一定风险——如果不熟悉 Linux 内核管理，建议先备份快照再操作。

安装完成后按提示重启，再次 SSH 连进去验证状态。

## 哪条线路值得加速，哪条加速意义不大

说句实话：**不是所有搬瓦工套餐加速效果都一样明显。**

- **KVM 套餐（普通线路）**：这档套餐走的是普通直连或 CN2 GT 线路，晚高峰本来就会抖，加速能补一些，但线路本身的瓶颈在那里，效果有上限。加速装了比不装强，但不要期望值太高。

- **CN2 GIA-E 套餐**：走的是 CN2 GIA 全程专线，本身线路质量就好，加速装了之后能把带宽利用率拉满，效果相对明显。**这档套餐最值得配合一键加速用。**

- **香港 CN2 GIA 套餐**：延迟 30-60ms，线路顶级，丢包极少，加速的边际收益相对小一些。不装也能用得很舒服，装了进一步优化带宽效率。

简单说：线路越差，加速的收益越高；线路越好，加速更像锦上添花。

## 常见问题

**Q：搬瓦工 KVM 一键加速之后效果怎么验证？**

SSH 登录后跑 `lsmod | grep bbr`，看到 `tcp_bbr` 说明 BBR 类加速已启动。如果装的是锐速，运行 `/appex/bin/lotServer.sh status` 查看运行状态。验证完之后可以用测速脚本对比加速前后的下载速度。

**Q：BBRplus 和 BBR 区别在哪？**

两者底层逻辑相似，BBRplus 在高丢包环境下的带宽占用更激进，理论上网络状况越差 BBRplus 的优势越明显。线路质量本来就好的话，两者体感差距不大。

**Q：锐速（Lotserver）适合哪些情况用？**

锐速是商业方案，对某些内核版本有要求，配置稍微比 BBR 系列复杂。适合对速度有极端要求、且愿意花时间折腾的用户。普通建站或日常用途，BBRplus 就够用了。

**Q：KVM 套餐和 CN2 GIA-E 套餐装完加速后，速度差距大吗？**

这个没有办法用一个数字回答，因为主要取决于线路本身。装了加速之后，CN2 GIA-E 在洛杉矶机房到国内的延迟大约在 130-160ms，晚高峰丢包很少；KVM 套餐走的是普通线路，延迟可能差不多，但晚高峰波动比 CN2 GIA-E 更明显。对于有带宽要求的场景，差距会更直观。
